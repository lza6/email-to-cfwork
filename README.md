
# 📬 从零到一：部署基于 Cloudflare 的大模型邮件 AI 看板

这是一个基于 Cloudflare Workers + KV 数据库 + DeepSeek（或其他大模型）构建的自动化邮件总结看板。

## 🩸 踩坑实录：我们为什么最终选择了这套架构？

在打造这个看板的过程中，我们经历了多次架构推翻与重构。如果你想自己修改代码，请务必先了解我们踩过的这 3 个致命大坑：

1. **死于“流式输出”的 KV 限流 (524 Timeout)**
   - **坑**：最初我们想做打字机效果，大模型吐一个字，我们就往 Cloudflare KV 里存一个字。
   - **真相**：Cloudflare KV 的物理限制是**同一键值每秒最多写入 1 次**。高频写入不仅会导致 API 被限流封禁，还会引发严重的 524 超时错误，进而触发邮件服务器疯狂重试，瞬间刷爆额度。
   - **解法**：采用 `ctx.waitUntil()` 挂起后台异步任务。瞬间接收邮件并存入 KV（耗时 < 50ms），后台花 30 秒慢慢请求大模型，完成后**一次性**写入 KV。前端采用 3 秒静默轮询获取结果。

2. **死于 URL 编码的“永远在思考”**
   - **坑**：面板上的邮件点击展开后，状态永远卡在“⚡ AI 思考中...”，哪怕后台其实已经处理完了。
   - **真相**：邮件的原生 ID 通常带有尖括号（如 `<tencent_80118...>`）。当前端发起请求时，浏览器会自动将 `<` 编码为 `%3C`。后端拿着 `%3C` 去 KV 库里找，当然找不到！
   - **解法**：在后端的 API 路由中，必须使用 `decodeURIComponent()` 进行严格解码。

3. **死于 `atob()` 的中文乱码 (`ä»£...`)**
   - **坑**：为了提取纯净正文，我们解析了邮件的 Base64 源码。但遇到 QQ、网易等国内邮箱时，中文全部变成了 `ä»£...` 这种火星文。
   - **真相**：JavaScript 原生的 `atob()` 函数天生不支持 UTF-8 编码的中文！
   - **解法**：引入 `TextDecoder('utf-8')`，将 Base64 转换为二进制流后进行标准 UTF-8 解码。

## 🚀 极其详细的部署教程 (Step by Step)

### 第一阶段：准备 Cloudflare 基础设施

**1. 创建 KV 数据库（AI 的记忆中枢）**
- 登录 Cloudflare，左侧菜单找到 **存储与数据库 -> KV**。
- 点击 **创建命名空间**。
- **名称必须精确填写**：`MAIL_SUMMARY_KV`（代码里写死了这个名字）。点击添加。

**2. 创建 Worker 脚本（AI 的大脑）**
- 左侧菜单找到 **计算 -> Workers 和 Pages**。
- 点击 **创建应用程序 -> 创建 Worker**。
- 名字随便起（比如 `ai-mail-bot`），点击 **部署**。
- 部署后，点击 **编辑代码**，将本项目中最终版的 `worker.js` 代码全部粘贴进去，覆盖原有内容，再次点击右上角的 **部署**。

**3. 绑定数据库与密钥（给大脑接上电源）**
- 退回到该 Worker 的总览页面。
- **绑定 KV**：点击 **设置 -> 绑定**。点击添加绑定，选择“KV 命名空间”。**变量名称**必须填 `MAIL_SUMMARY_KV`，右侧下拉框选择你第一步创建的数据库。
- **配置环境变量**：切换到 **设置 -> 变量**，添加以下三个变量（缺一不可）：
  - `API_URL`：填写大模型接口（如 `https://api.deepseek.com/v1/chat/completions` 或者 Nvidia 提供的接口）。
  - `API_KEY`：填写你的模型密钥（**强烈建议点击旁边的“加密”按钮**）。
  - `MODEL_NAME`：填写模型具体版本（如 `deepseek-chat` 或 `deepseek-ai/deepseek-v4-pro`）。

---

### 第二阶段：配置电子邮件路由（最容易卡死的环节！）

这是最关键的一步，由于 Gmail 的安全机制，我们**必须分两步走，不能一步到位**。

**第 1 步：先让 Cloudflare 成为 Gmail 的“转发站”**
- 在 Cloudflare 左侧菜单点击 **电子邮件 -> 电子邮件路由**。
- 切换到 **目标地址** 标签，添加你的个人常用 Gmail（此时 Cloudflare 会发一封验证信到你的 Gmail，去邮箱点击验证）。
- 切换到 **路由规则** 标签，点击 **创建地址**。
  - **自定义地址**：比如输入 `ai`，后缀选你的域名（例如 `ai@tfai.indevs.in`）。
  - **操作**：⚠️ **此处非常重要！一定要先选择“发送到电子邮件”**。
  - **目标**：选择你刚才验证过的个人 Gmail。
  - 点击保存。

**第 2 步：去 Gmail 获取“自动转发验证码”**
- 登录你的网页版 Gmail，点击右上角齿轮 ⚙️ -> **查看所有设置 -> 转发和 POP/IMAP**。
- 点击 **添加转发地址**，输入你刚才创建的自定义地址（`ai@tfai.indevs.in`）。
- 此时 Gmail 会向这个地址发一封包含验证码的邮件。**因为你在上一步设置了“发送到电子邮件”，所以这封带验证码的邮件会乖乖转回到你的 Gmail 收件箱里！**
- 拿到验证码，填入 Gmail 设置中完成验证。
- **设置过滤规则**：强烈建议点击“创建过滤器”，设置只转发特定条件的重要邮件，而不是勾选全局的“将接收到的邮件副本转发给...”（不然你的广告邮件会把 AI 刷爆）。

**第 3 步：切断转发，正式接入 Worker（点睛之笔）**
- 回到 Cloudflare 的 **电子邮件路由 -> 路由规则**。
- 找到你刚才建的那个自定义地址（`ai@tfai.indevs.in`），点击 **编辑**。
- ⚠️ **将“操作”从“发送到电子邮件”修改为“发送到 Worker”**。
- 在弹出的下拉框中，选择你创建的 Worker（`ai-mail-bot`）。
- 点击保存。

**至此，完美闭环！** 以后的邮件将直接进入代码处理，不会再去你的收件箱里套娃了。
